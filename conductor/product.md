# Product Definition — JobSync

**Project Name:** JobSync
**Type:** Brownfield (existing application, v1.1.3)

## Description

JobSync is a self-hosted job application tracker with EURES/ESCO integration for
automated European job discovery, applicant CRM, and AI-assisted resume matching.

## Problem Statement

Job seekers managing many applications across multiple boards struggle to track
status, discover relevant European vacancies, and keep recruiter relationships
organised — while staying in control of their own data. Cloud SaaS trackers leak
PII to third parties and lock data behind subscriptions. JobSync solves this by
being **self-hosted** (own data, own server), **automation-driven** (scheduled
EURES/Arbeitsagentur/JSearch discovery), and **GDPR-first** (PII redaction before
any cloud-AI transfer, local Ollama as the privacy-preserving default).

## Target Users

1. **Individual job seekers** (primary) — self-host JobSync to track applications,
   discover EU vacancies, manage recruiter contacts, and match their resume to roles.
2. **Privacy-conscious users** — want full ownership of application + PII data, no
   third-party SaaS, optional local-only AI (Ollama).
3. **Power users / tinkerers** — run automations, connect modules (job boards, AI
   providers, enrichment), customise notification channels.

## Key Goals

1. **Data ownership & GDPR compliance** — all PII stays local unless the user opts
   into a cloud provider; egress is redacted; audit trails for Art. 5(2).
2. **Pluggable discovery & enrichment** — a Connector/Module ACL architecture so new
   job boards, AI providers, and data-enrichment sources drop in via a Manifest with
   zero core changes.
3. **Application lifecycle visibility** — Kanban board, staging deck, CRM timeline,
   and multi-channel notifications give a single source of truth for every application.

## Core Domains (Bounded Contexts)

- **Job Aggregate** — Job + Notes + Tags + Status (the application record).
- **Automation Aggregate** — scheduled job-search configs + runs + discovered vacancies.
- **Profile Aggregate** — Profile + Resumes + Sections + Contact Info.
- **Person Aggregate (CRM)** — Person + CrmInterviews + JobContacts + CrmTask/Note targets.
- **Connector contexts** — Job Discovery (EURES, Arbeitsagentur, JSearch), AI Provider
  (Ollama, OpenAI, DeepSeek), Data Enrichment (Logo.dev, Google Favicon, Meta Parser),
  Reference Data (ESCO, Eurostat NUTS, GeoCodes, Public Holidays).
- **Notification dispatch** — multi-channel (in-app, webhook, email, push).
- **Public API v1** — REST Open Host Service over the data layer.
