## Summary

Two related improvements (ROADMAP 3.11, code-verified `docs/ROADMAP.md:1733`):
- **Phase 1 ‚ÄĒ Stale-Session Guard:** a JWT can carry a user id that no longer exists
  after a DB reset/migration ‚Üí every write fails with a cryptic FK error (P2003). Guard
  `getCurrentUser()` so a stale id returns `null` (= "not authenticated"), and the
  dashboard shows a clear re-login banner instead of a cryptic crash.
- **Phase 2 ‚ÄĒ Form State Persistence (`usePersistedForm`):** a `react-hook-form` wrapper
  that auto-saves form state to `localStorage` (debounced), so an interrupted session
  (incl. a stale-session re-login) restores in-progress form input.

## Acceptance Criteria

- [ ] `getCurrentUser()` verifies the JWT user id still exists in the DB (with an

## Technical Notes

- Phase 1 cache must invalidate cleanly (60s TTL); the DB check must not add a query per
  request when the cache is warm.
- `usePersistedForm` localStorage keys are per-form, TTL-bounded, and cleared on submit ‚ÄĒ
  do not persist PII longer than the 30-min TTL (GDPR-minded).
- Mirror `useKanbanState` / `useStagingLayout` for the storage idiom (SSR-safe, hydrate on mount).
