## Architecture Highlights

- **Connector/Module ACL pattern** ‚Äî App ‚Üî Connector (shared domain layer) ‚Üî Module
  (pluggable, Manifest-described). Connector types: `job_discovery`, `ai_provider`,
  `data_enrichment`, `reference_data`.
- **Manifest-driven UI** ‚Äî AutomationWizard is a generic rendering engine consuming
  manifests (`connectorParamsSchema`, `searchFieldOverrides`, widget registry).
- **Module Lifecycle Manager** ‚Äî unified `ModuleRegistry`, credential PUSH resolution,
  health monitoring, degradation rules.
- **CRM core** ‚Äî Person aggregate, polymorphic targeting (Twenty CRM pattern), CRM cron.
- **Multi-channel notifications** ‚Äî ChannelRouter (in-app, webhook, email/SMTP, web push/VAPID).
- **EURES/ESCO integration** ‚Äî EU job discovery + occupation taxonomy.

## Infrastructure / Deployment

- **Self-hosted / Docker** ‚Äî Docker volume convention `/data` for persistent state.
- **NixOS dev environment** ‚Äî `devenv.nix` (writable store) OR `./scripts/*.sh` helper
  scripts (read-only store / VMs). Prisma engines auto-patched for NixOS.

## Package Manager & Tooling

- **bun** is the package manager (NOT npm/yarn). Use `bun add`, `bun run`.
- **Jest** + Testing Library ‚Äî unit/component/integration tests (`__tests__/*.spec.ts`).
- **Playwright** + system Chromium ‚Äî E2E (`e2e/`, smoke + crud projects).
- **Knip** ‚Äî dead-code detection (`bun knip`, config `knip.ts`).
- **Build/typecheck:** `source scripts/env.sh && bun run build` (zero type errors required).
  Stop the dev server before running tsc/build.
- **Tests:** `bash scripts/test.sh` (uses system Node.js, not bun).
