# Product Guidelines — JobSync

## Voice and Tone

- **UI text & docs:** concise and direct, professional but approachable. No jargon
  in user-facing strings.
- **Every user-visible string is translated** — EN, DE, FR, ES (see Workflow → i18n).
  Default authoring locale is EN; DE/FR/ES are mandatory peers, not afterthoughts.
- **Error messages shown to users** (toasts, `result.message`) MUST be i18n keys,
  never hardcoded English.

## Design Principles

1. **User data sovereignty first** — self-hosted, GDPR-by-design. PII never leaves
   the instance unredacted. Local Ollama is the privacy-preserving default; cloud AI
   is opt-in and minimised.
2. **Best-effort, non-blocking enrichment** — never surface enrichment failures to
   users; fall back to placeholders.
3. **Pluggability over hardcoding** — modules self-describe via Manifests; UI is a
   generic rendering engine (no `if (module === "x")` branches).
4. **Security as a contract, not a feature** — IDOR ownership on every query (ADR-015),
   `"use server"` export discipline (ADR-019), SSRF validation on every outbound fetch.
5. **Specification-driven domain rules** — complex behaviour is pinned by Allium specs
   in `specs/*.allium` before code is written; specs are the single source of truth.
6. **Accessibility** — WCAG-conscious; motion-reduce respected; keyboard + screen-reader
   paths for interactive surfaces (deck, sheets, comboboxes).

## Domain Language (Ubiquitous Language)

Use exact domain terms in code, UI, specs, commits, docs. Key terms:
`DiscoveredVacancy` (not "scraped job"), `Connector` (not "scraper"),
`Module` (not "API/service"), `Automation` (not "cron job"), `Person` (CRM entity,
not legacy "Contact"), `ActionResult<T>` (not `Promise<any>`).
Full glossary in `/CLAUDE.md` → "Ubiquitous Language".
